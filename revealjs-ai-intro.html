核心判断

reveal.js 不是 “另一个 PPT”
而是把演示文稿放进了 Web + AI 的生产体系

传统 PPT 方式
  • 很多时间耗在排版、复制页面、调动画
  • 复杂视觉与交互效果成本高
  • 协作与版本管理不顺手
  • AI 只能帮写字，排版仍得手工收尾
reveal.js 方式
  • 结构由 HTML 定义，样式由 CSS 统一控制
  • 动画、嵌套内容、代码展示、媒体嵌入更灵活
  • 天然适合 Git 与文本协作
  • AI 可以直接生成整份 deck 的内容与骨架
如果 PPT 更像“做文件”，那 reveal.js 更像“做一个会演讲的网页”。说白了，就是从手工排版转向可编排系统。
适合产品汇报、技术分享、课程、路演、方案陈述02

效果示范：Fragments

一页之内，按顺序讲，不要一上来全盘托出

官方 fragments 文档的意思很简单：给元素加上 class="fragment"，它就会在当前页里按步骤依次出现。

  • 第一步：先说问题——为什么 PPT 会越改越乱
  • 第二步：再说方法——结构、样式、动画分离
  • 第三步：再说效率——AI 可直接生成初稿与页面结构
相比复制三页近似页面，Fragments 更像真正的“演讲推进”05

效果示范：Fragments

不仅能分步出现，还能分步强调

  • 重点 1：减少重复页面
  • 重点 2：控制信息释放节奏
  • 重点 3：让 AI 更容易生成“演讲式”结构

导出与交付

现代演示方式，不等于放弃传统交付

Web

在线演示

静态部署即可：Vercel、GitHub Pages、内网 Web 服务器都可以。

PDF

PDF 导出

官方文档说明：在 URL 后追加 ?print-pdf，再用 Chrome/Chromium 打印为 PDF。

Git

可管理版本

因为本质是文本文件，所以可以做版本对比、回溯、多人协作，这点对团队非常重要。

官方 PDF 导出要点：打开带 ?print-pdf 的链接 → 浏览器打印 → Landscape → Margins None → 开启 Background graphics → 保存为 PDF。如此便能兼顾现代展示与传统归档。
这点非常适合企业内部“现场汇报 + 事后发 PDF”双需求14

AI 工作流

为什么 reveal.js 特别适合和 AI 一起协作

Step 01

给定目标

明确主题、对象、页数、风格、是否需要动画、是否偏管理汇报还是技术分享。

Step 02

先出大纲

要求 AI 先给逻辑结构，而不是一上来就做页面装饰。

Step 03

生成 HTML + CSS

让 AI 直接输出 reveal.js 单文件 HTML，随后按需要迭代样式。

Step 04

反复精修

结合预览截图，让 AI 逐轮优化字数、留白、卡片密度与动画强度。

推荐提示词模板：
请为我制作一份 reveal.js 单文件 HTML 幻灯片，主题是「向还在使用 PPT 的人介绍 reveal.js」。受众是企业内部汇报场景中的管理者与知识工作者。整体风格稳重、简洁、专业、像领导汇报，不要科技感过强。使用 reveal.js v5 CDN；适度使用 fragments、auto-animate、vertical slides、speaker notes、代码高亮与 PDF 导出说明；页面少堆字，多用卡片与层次。
AI 擅长结构化输出，而 reveal.js 恰好是可结构化描述的15

落地方法

一个非常实用的 AI + reveal.js 制作流程

workflow

1. 先做人群导向的大纲

例如：对领导讲价值、对部门讲计划、对技术团队讲实现。

2. 再做页面结构

哪些页面是两栏，哪些页面是卡片，哪里需要分步 reveal。

3. 最后统一视觉语言

颜色、字体、圆角、网格、阴影、页脚说明、讲者备注一并定型。

Tips

几个很重要的建议

  • 先要大纲，再要页面，不要一步乱炖
  • 要求 AI 减字、增强层次、留足留白
  • 明确写“偏企业汇报，不要炫技网页风”
  • 每一轮只优化一个维度：内容 / 视觉 / 动效 / 备注
Role

正确分工

AI 负责提速、起稿、统一风格；人负责判断、删减、取舍、定调。让 AI 做副驾，而不是抢方向盘。

不是让 AI 替代汇报，而是让它把机械劳动拿走16

风格可以非常克制，不一定非得赛博朋克17

Final takeaway

如果你还在用 PPT 做“网页时代的汇报”
那 reveal.js 值得成为你的 下一代演示底座

它开源、成熟、可部署、可导出 PDF、可接入 AI，而且可以做得像企业内部高质量汇报一样稳重克制，并不一定要张牙舞爪。真正的关键，不在于特效有多酷，而在于表达是否更清楚、生产是否更高效。
